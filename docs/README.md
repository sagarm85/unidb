# unidb Documentation

- [REST_API.md](REST_API.md) — full HTTP route reference for the optional
  `unidb-server` binary (M5): every route's payload and response shape,
  auth model, error codes, and known limitations.
- [backlog/](backlog/) — saved plans for future work that hasn't started
  yet (e.g. `phase2_sql_capability_expansion.md`). These are durable,
  git-tracked references — unlike Claude Code's own ephemeral plan-mode
  file, which gets overwritten by the next plan.

For design decisions, milestone status, and architecture, see the
project-root `CLAUDE.md`, `PROGRESS.md`, and `MEMORY.md`.
